# workflow-bancaire-simplifie
Simulation pédagogique d’un workflow transactionnel bancaire Front → Middle → Back en Java, illustrant la création, validation et règlement d’une transaction.


# RESUME
Ce mini-projet illustre la chaîne de traitement d’une transaction financière au sein d’une banque, depuis sa création par le Front Office, sa validation par le Middle Office, jusqu’à sa confirmation et comptabilisation par le Back Office.
Aucun moteur financier réel, uniquement une logique métier simple et formatrice.
Idéal pour comprendre les rôles séparés et la logique post-trade.
